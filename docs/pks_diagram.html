Consistency (C)

Toda leitura retorna o dado mais recente ou um erro. PKS sacrifica consistência forte para não bloquear queries.

Availability (A)

Toda requisição recebe uma resposta (mesmo desatualizada). Queries BM25 nunca bloqueiam — retornam resultados do índice atual.

Partition Tolerance (P)

Sistema continua operando mesmo com partições (ex: Ollama offline). Graceful degradation para modo BM25-only sem quebrar o contrato MCP.

Resultado AP: Pequenas defasagens de índice são aceitáveis. Reparos ocorrem assincronamente via pipeline de eventos. Prioridade: estar sempre disponível para o LLM.
