---
description: Verifique visualmente se tudo ficou correto
---

O objetivo aqui é validar se o último trabalho realizado atende ao que foi pedido. NUNCA peça para o usuário validar nada visualmente por você, essa é a principal função deste workflow.

* Essa verificação deve ser feita visualmente, ou seja, olhando os resultados, não olhando o código.
* Utilize o navegador se for uma solução web.
* Utilize o terminal se for uma solução de linha de comando.
* Utilize qualquer ferramenta compatível com que valide visualmente o resultado.

Verifique se durante a execução do trabalho não quebrou visualmente outras partes das telas que podem estar relacionadas ao trabalho realizado. Se quebrar, corrija o problema sem pedir mais nada ao usuário. Depois volte para o início deste workflow e execute novamente.

Se o resultado não atender ao que foi pedido, corrija o problema sem pedir mais nada ao usuário. Depois volte para o início deste workflow e execute novamente.

Se o resultado atender ao que foi pedido, avise o usuário, mostre uma imagem do resultado como evidência e conclua o trabalho.
